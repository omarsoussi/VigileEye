Clean Architecture Summary

All four backends follow Clean Architecture with consistent layer separation:

  • api/ — HTTP/WS layer (routes, dependencies) — depends on application
  • application/ — Use cases, DTOs, orchestration — depends on domain
  • domain/ — Entities, repository interfaces, value objects — no dependencies
  • infrastructure/ — DB, external services, config — implements domain interfaces

Dependencies flow inward: api → application → domain ← infrastructure
